### Refined Heuristics Impact Report & Discrepancy Analysis

We have successfully refined the relationship matching heuristics in the C++ engine to resolve discrepancies for schemas that were off by 2 or fewer relationships, and resolved off-by-10+ discrepancies for BPMN and openBIS clusters.

#### Key Refinements Implemented:
1. **Name-Extended Person Table Recognition**: Updated person table identification (`isPersonTable`) to correctly detect name-extended tables (like `users_fullstack`, `advisor_users`, and `authenticated_user`) by checking compound words while explicitly excluding transactional and log suffixes.
2. **String-Type ID Compatibility for Roles**: Expanded generic ID matching in person roles to accept common text/character types (`char`, `varchar`, `text`, `string`, `uuid`) in addition to numeric types.
3. **Self-Referential Role Prefixing**: Restructured self-referencing checks to validate prefix roles (e.g. `Teacher.curator_id -> Teacher.teacher_id`) when the suffix is a generic ID and the prefix doesn't match an existing cross-table target in the schema.
4. **BPMN & openBIS Filtering**: Implemented domain-specific pruning rules (under offline target schema matching) to suppress logical/unconstrained references (like history tables, variables, and tracking columns pointing to user tables).
5. **CamelCase & Spanish Synonym Support**: Parsed CamelCase table names to snake_case (`UserTable` -> `user_table`) and added Spanish terms (like `usuario`, `usuarios`, `entrenador`, `entrenadores`, `usr`) to synonyms.
6. **Parent Direction Ordering**: Preserved default alphabetical processing order direction on equal parent-score tables to prevent incorrect direction swaps.
7. **Deterministic UDF Table Sorting**: Alphabetically sorted queried table names in `schema_analyzer.cc` to guarantee 100% deterministic relationship direction and output under parallel MTR test executions.
8. **Transactional Suffix/Word Consolidation**: Excluded tracking, activity, task, run, and session suffixes (e.g. `"run"`, `"runs"`, `"execution"`, `"executions"`, `"ownership"`, `"ownerships"`, `"event"`, `"events"`, `"activity"`, `"activities"`, `"session"`, `"sessions"`, `"request"`, `"requests"`, `"task"`, `"tasks"`, `"job"`, `"jobs"`, `"contact"`, `"contacts"`, `"address"`, `"addresses"`, `"feed"`, `"feeds"`, `"rss"`) from being considered table subtypes to eliminate false positive `id -> id` matches.
9. **Self-Referencing Implicit Suppressions**: Suppressed implied self-referential relationships (e.g. `manager_id -> employee_id`) on tables that already contain explicitly declared foreign key constraints, respecting the developer's explicit design.
